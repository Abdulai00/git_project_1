heckpoint 1: Staging Is Not Committing
Stage specific files: git add tests/test_calculator.py
Commit first change: git commit -m "fix"
Stage second file: git add src/calculator.py
Commit second change: git commit -m "update"
Learning: I learned how to use the staging area to separate unrelated changes into distinct commits, even if they were worked on at the same time.

Checkpoint 2: Undo Without Erasing History
View history for ID: git log --oneline
Revert specific commit: git revert <commit-hash-of-BAD>
Learning: I learned that git revert is safer than deleting commits because it creates a new inverse commit, preserving the full project history while fixing the code.

Checkpoint 3: Moving a Commit (Cherry-Pick)
Switch to main: git switch main
Commit to main: git commit -m "Add is_positive helper function"
Undo commit on main: git reset --hard HEAD~1
Move to feature: git switch feature/120403934
Apply commit here: git cherry-pick <hash-from-main>
Resolve Conflict: Edit file, git add src/validator.py, git cherry-pick --continue
Learning: I learned how to teleport a commit from one branch to another and how to handle conflicts that arise when the same lines are modified in both places.

Checkpoint 4: Recovering Lost Work (Reflog)
Delete branch: git branch -D experiment/120403934
Find lost hash: git reflog
Restore commit to a branch: git branch recovered/120403934 <hash-from-reflog>
Merge into feature: git merge recovered/120403934
Learning: I learned that almost nothing is ever truly deleted in Git; reflog acts as a safety net to recover work that seems lost after a branch deletion.

Checkpoint 5: Syncing with Upstream
Add remote: git remote add upstream <url>
Fetch latest: git fetch upstream
Update main: git switch main -> git merge upstream/main --ff-only
Rebase feature: git switch feature/120403934 -> git rebase main
Push updates: git push --force-with-lease origin feature/120403934
Learning: I learned how to synchronize my local work with the team's latest changes and how to use rebase to keep a linear, clean history.

Checkpoint 6: Interactive Rebase
List commits: git log main..HEAD --oneline
Start rebase: git rebase -i main
Rebase Actions: Used reword for messages, fixup/squash to combine fix and update, and reorder to move the BAD and Revert commits together.
Skip empty commit: git rebase --skip
Learning: I learned how to curate my commit history to make it readable for reviewers by squashing tiny fixes and rewriting vague messages.
