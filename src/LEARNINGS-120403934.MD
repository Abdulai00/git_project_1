Checkpoint 1: Staging Is Not Committing
git add tests/test_calculator.py

git commit -m "fix"

git add src/calculator.py

git commit -m "update"

Learning: I learned how to use the staging area to separate unrelated changes into distinct commits, even if they were worked on at the same time.

Checkpoint 2: Undo Without Erasing History
git log --oneline

git revert <commit-hash-of-BAD>

Learning: I learned that git revert is safer than deleting commits because it creates a new "inverse" commit, preserving the full project history while fixing the code.

Checkpoint 3: Moving a Commit (Cherry-Pick)
git switch main

git commit -m "Add is_positive helper function"

git reset --hard HEAD~1

git switch feature/<your-id>

git cherry-pick <hash-from-main>


Learning: I learned how to "teleport" a commit from one branch to another and how to handle conflicts that arise when the same lines are modified in both places.

Checkpoint 4: Recovering Lost Work (Reflog)
git branch -D experiment/<your-id>

git reflog

git branch recovered/<your-id> <hash-from-reflog>

git merge recovered/<your-id>

Learning: I learned that almost nothing is ever truly deleted in Git; reflog acts as a safety net to recover work that seems lost after a branch deletion.

Checkpoint 5: Syncing with Upstream
git remote add upstream <url>

git fetch upstream

git switch main -> git merge upstream/main --ff-only

git switch feature/<your-id> -> git rebase main

git push --force-with-lease origin feature/<your-id>

Learning: I learned how to synchronize my local work with the team's latest changes and how to use rebase to keep a linear, clean history.

Checkpoint 6: Interactive Rebase
git log main..HEAD --oneline

git rebase -i main

Learning: I learned how to "curate" my commit history to make it readable for reviewers by squashing tiny fixes and rewriting vague messages.
